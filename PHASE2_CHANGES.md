# 生活帮第二阶段改造说明

本阶段把“工程师报价 → 正式项目 → 文件交付 → 需求变更 → 验收/争议”从演示字段升级为可追溯业务记录。

## 已完成

### 1. 报价版本

- 新增 `quote_versions`
- 报价首次提交自动生成 V1
- 工程师可以创建 V2、V3 等新版本
- 每个版本保留价格、工期、交付内容、不包含内容、付款节点、修改次数、售后和更新说明
- 已接受、已撤回、已拒绝的报价不能继续修改
- 历史数据库报价会在迁移中自动补建 V1

### 2. 正式需求与双方确认

- 新增 `project_requirements`
- 接受报价时自动生成正式需求 V1
- 需求方接受报价视为需求方初次确认
- 工程师必须在项目页确认正式需求与合作条款
- 双方均确认后，项目才能进入付款状态
- 旧项目通过迁移自动回填正式需求版本

### 3. 项目文件中心

- 新增 `project_files`
- 支持文档、设计、交付、测试、协议和其他文件分类
- 可关联具体里程碑
- 同一文件可以上传新版本，旧版本自动标记为“已替代”
- 正式交付文件不能直接删除
- 单文件限制 8MB
- 项目文件不再通过公开 `/uploads/projects/*` 访问
- 仅项目成员可以申请 5 分钟有效的签名下载链接

### 4. 项目变更单

- 新增 `project_changes`
- 需求方和工程师都可以发起变更
- 变更记录内容、原因、金额变化、工期变化和交付物影响
- 变更提出人不能替对方确认
- 对方同意后自动：
  - 更新项目金额
  - 更新预计完成日期
  - 生成新的正式需求版本
  - 将旧需求版本标记为“已替代”
- 变更可以拒绝或由提出人撤回

### 5. 验收记录

- 新增 `project_acceptances`
- 每次验收通过和要求修改均形成独立记录
- 里程碑验收改为数据库事务
- 重复验收不会重复创建项目订单或信用事件
- 最后一个里程碑通过后，项目、需求、订单和信用在同一事务内更新

### 6. 投诉与争议

- 新增 `complaints` 和 `complaint_evidence`
- 项目和里程碑可以发起投诉
- 发起后项目进入争议状态
- 被投诉方可以提交正式回应
- 双方可以补充证据说明
- 通知可以直接跳转到投诉详情
- “我的”页面增加投诉与争议入口

### 7. 数据迁移与种子数据

- 新增迁移 `drizzle/0003_spotty_captain_flint.sql`
- 自动回填已有报价版本
- 自动回填已有项目正式需求
- 更新种子数据，使演示报价包含版本记录
