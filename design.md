# 生活帮 App 界面设计文档

移动端竖屏(9:16)、单手操作优先,遵循 iOS HIG 与文档指定的"可信、温和、简洁、生活化"风格。

## 品牌与色彩

| 令牌 | Light | 用途 |
|------|-------|------|
| primary | #16A34A(温和绿) | 主行动、品牌、选中态 |
| accent | #0D9488(蓝绿) | 工程师/技术相关 |
| action | #F97316(暖橙) | 发布按钮、行动引导 |
| background | #F7F8F7 | 页面背景 |
| surface | #FFFFFF | 卡片 |
| foreground | #1A1D1B | 主文字 |
| muted | #6B7280 | 次要文字 |
| success/warning/error | 绿/黄/红 | 状态色 |

原则:禁止大量互相竞争的彩色卡片;状态优先于装饰;AI 建议与人工确认明显区分(AI 内容带"AI 整理"标识与浅色底)。

## 屏幕清单

### 身份切换的三套 Tab
- 用户端 Tabs:首页 home / 发现 discover / 发布 publish(中央橙色圆形突出) / 消息 messages / 我的 profile
- 工程师端(同一 Tab 结构下按当前身份渲染):工作台 / 需求大厅 / 项目 / 消息 / 我的
- 商家端:工作台 / 附近询价 / 订单 / 消息 / 我的
- 实现:单一 (tabs) 组,Tab 内容根据 Zustand currentRole 切换渲染;Tab 标题与图标随身份改变。

### 主要屏幕
1. 登录页 login:手机号+验证码(演示环境固定验证码 123456)、协议勾选;未登录访问受限操作时跳转。
2. 完善资料 profile-setup:昵称、城市。
3. 首页(用户):顶部城市+搜索框"搜问题、工程师、服务或物品"+通知铃铛;AI 生活助手大卡片"遇到了什么问题?"(输入直达发布流程);快捷问题 chips;6 宫格快捷入口(发布需求/找工程师/附近服务/旧物出售/物品回收/免费赠送);推荐区块:附近需求、推荐工程师、附近旧物。
4. 发现页 discover:tabs 推荐/需求/工程师/旧物;需求卡(标题/类型/预算/位置/状态/响应数)、工程师卡(头像/职称/技能/评分/起步价/认证徽章)、旧物卡(图/名/价/成色/方式)。
5. 发布中心 publish:6 个入口卡片:我遇到了问题 / 我有一个产品想法 / 我需要专业服务 / 我想发布物品 / 我需要维修或回收 / 我想帮助别人(跳转赠送)。
6. 需求发布流程 needs/create(4 步):①描述问题(标题/描述/类型/城市/是否公开) ②AI 整理(调用 LLM 生成结构化:使用对象/场景/问题/期望效果/预算建议/推荐专业/风险提示,可编辑,失败可跳过) ③补充信息(分类/预算区间/期望时间/上门/远程/允许评论/允许报价) ④确认发布。
7. 需求广场 needs/index + 需求详情 needs/[id]:详情含描述、AI 结构化信息、状态时间线、"我也需要"、评论、方案/报价列表;创建者可关闭需求、查看报价并接受;工程师身份可提交方案/报价。
8. 报价比较 needs/[id]/quotes:多报价对比(总价/工期/交付内容/修改次数/售后),接受报价→弹确认→创建项目。
9. 项目列表 projects + 项目详情 projects/[id]:状态机徽章、里程碑列表(开始/提交交付/验收/要求修改)、项目动态;验收通过→结算提示,全部完成→项目 completed、需求 solved。
10. 物品发布 items/create:物品档案(名称/分类/成色/功能状态/描述/图片)+流转方式(一口价/接受报价/置换/赠送/回收)+定价 → 发布 listing。
11. 旧物市场 listings + 详情 listings/[id]:按方式展示操作(立即购买/出价/申请领取/换物);卖家可接受报价(创建订单)。
12. 回收 recycling:提交回收询价(物品/类别/照片/期望)→商家报价列表→选择报价→(演示)检测→确认成交。
13. 订单 orders + 详情 orders/[id]:状态流转(待确认/待支付/已支付/待交付/待验收/已完成),模拟支付按钮,确认收货,评价入口。
14. 消息 messages:分类 聊天/项目/订单/系统;会话列表+聊天页 messages/[id];通知列表。
15. 我的 profile:头像/昵称/当前身份切换/信用徽章/城市;分组入口:我的需求、我的项目、我的物品、我的订单、收藏、信用中心 credits、成为工程师(认证申请)、商家入驻、设置。
16. 工程师工作台:接单开关、统计(响应/完成率)、新需求推荐、我的报价、进行中项目。
17. 工程师认证 engineer-apply:姓名/城市/专业方向/从业年限/介绍/技能标签(演示即时通过→基础认证)。
18. 商家工作台:今日询价、回收报价管理、订单概览。
19. 信用中心 credits:信用等级(良好/正常/需关注/高风险)、信用事件列表、评价列表。
20. 评价页 orders/[id]/review:按维度打分(专业能力/需求理解/交付质量/准时/沟通)+文字。

## 关键用户流
1. 发布需求:首页 AI 卡片输入 → needs/create 带入描述 → AI 整理确认 → 补充信息 → 发布 → 需求详情(published)。
2. 工程师接单:切换工程师身份 → 需求大厅 → 需求详情 → 提交报价 → 用户在报价比较页接受 → 自动创建项目、其他报价 not_selected、需求 project_created。
3. 项目交付:工程师开始里程碑 → 提交交付 → 用户验收(通过/要求修改) → 全部通过 → 项目完成、需求 solved、双方可评价。
4. 旧物出售:发布物品(一口价+接受报价) → 买家出价 → 卖家接受 → 订单创建 → 模拟支付 → 确认交付/收货 → 完成 → 评价。
5. 回收:用户提交询价 → 商家(演示商家)报价 → 用户选择 → 成交订单。

## 交互规范
- 每页一个主操作按钮(底部通栏,主色);高风险操作(接受报价/关闭需求/确认收货)用 ConfirmDialog 明确文案。
- 状态用 StatusBadge 统一颜色:草稿灰/待审核黄/已发布绿/进行中蓝绿/已完成绿/已关闭灰/争议红。
- AI 生成内容统一带"AI 辅助整理,请确认后再发布"提示条。
- 空状态给引导性文案+行动按钮。
- 列表 FlatList + 下拉刷新。
