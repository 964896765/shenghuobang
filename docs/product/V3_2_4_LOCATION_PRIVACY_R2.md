# V3.2.4-R2 前台位置与隐私规则

1. App 冷启动、登录和首页加载不得触发系统位置权限请求。
2. 只有发现、搜索和工程师列表的用户主动按钮可以在用途说明确认后请求前台权限。
3. 不声明或使用后台位置权限，不收集轨迹、不做围栏、不在通知中附带坐标。
4. 当前设备精确坐标只在客户端内存和单次 TLS API 请求中使用；不得写入普通日志、错误、通知或审计详情。
5. 服务端偏好表把设备坐标四舍五入到小数点后两位，约为 1 公里级；手动地区不保存坐标。
6. 公开发现、搜索和附近接口只返回 `distanceKm`（整数近似值）、`distanceLabel` 或地区文字，不返回任何用户坐标字段。
7. 位置偏好按 `userId` 唯一且只能由本人更新；写入与追加式审计处于同一事务。
8. 无权限、永久拒绝、系统定位关闭、超时、弱网、数据库失败和无资源坐标都必须保留非位置核心功能。
9. 手动地区按账号缓存；退出登录清理所有账号相关本地位置缓存，防止换账号残留。
10. 前后台切换回到 active 时只刷新系统权限状态，不自动重新请求或获取位置。

本规则不授权生产位置画像、精确地址公开、后台定位或第三方位置 SDK。
