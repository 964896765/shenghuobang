# UI / UX 统一标准

## 1. 说明

- 本标准基于当前主题、公共组件和主要页面提取，不重新设计一套全新视觉品牌。
- 以下数值为待用户批准的 V3.2.4 拟冻结值；这不代表未来不能进行品牌重设计。
- 本标准服务于 V3.2.4 审查，不直接等同于已全部实现。

## 4. 页面骨架

### 页面头部

- 二级页面统一使用标题 + 返回。
- 头部高度与标题层级应稳定，不因副标题或右侧按钮出现而跳动。

### 返回按钮

- 左上返回必须可见、可点。
- 触控区域不小于 `44x44 dp`。
- 搜索页、输入页遵循“先收键盘，再返回页面”。

### 底部导航

- 固定 5 个主入口：首页、发现、发布、消息、我的。
- 当前选中态、未读角标、图标容器和文案要垂直对齐。
- 最后一项内容不得被底部导航和安全区叠加遮挡。

## 5. 按钮与表单

### 主按钮

- 用于登录、保存、发布、提交、确认。
- 建议冻结高度 `48`、圆角 `12`、正文 `16`。
- 保留 Android 按压缩放反馈，但必须避免空 `transform`。

### 次按钮

- 用于取消、返回列表、次级筛选、查看更多。
- 视觉权重低于主按钮，但边界要清楚。

### 危险按钮

- 用于删除、关闭、拒绝、取消等高风险动作。
- 必须配合确认弹窗。

### 输入框

- 标签、占位、错误提示齐备。
- 单行输入统一为 `48` 高。
- 搜索框总高度统一为 `44`。

### 键盘弹出策略

- 键盘弹出时当前输入字段必须可见；主操作必须可通过滚动或固定操作区到达，不得被键盘永久遮挡。
- 搜索页优先保证输入框、热门词、结果区不闪动重叠。
- 多字段表单必须支持自动滚动到聚焦项。

## 6. 页面状态

### 空状态

- 区分“暂无数据”和“搜索/筛选后无结果”。

### 加载状态

- 首屏不能长时间白屏。
- 局部加载尽量不阻塞整个页面。

### 错误状态

- 明确区分网络失败、服务异常、无权限。
- 必须提供重试入口。

### Toast

- 仅承担轻量反馈，不替代页面级错误说明。

### 确认弹窗

- 标题、正文、确认文案必须明确。
- 当前弹窗现状为 `radius16 padding20 maxWidth360`，建议保留。

## 7. 移动端交互

### Android 返回键

- 键盘打开时先收键盘。
- 弹窗打开时先关弹窗。
- 非根页面返回上一级，不直接退到桌面。

### 长页面滚动

- “我的”页、设置页、详情页最后一项必须完整露出。
- 长列表滚动位置和下拉刷新要可预期。

### 弱网反馈

- 必须告诉用户是网络问题还是服务未连接。
- 本地联调场景下允许提示检查 API 地址与当前网络。

### 权限拒绝

- 权限被拒绝后要给出原因与下一步引导。
- 不允许拒绝后长期空白或伪装为加载中。

## 8. 当前差异与备注

- 本文件冻结的是 V3.2.4 临时体验标准，不代表未来不能重新设计品牌。
- 当前实现若与本文件不一致，后续审查按本文件登记差距。
