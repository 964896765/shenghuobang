# 完整用户流程图

## 1. 说明

- 本文档描述“生活帮”当前需要审查的完整用户流程。
- “当前是否完整”用于表达产品完整度，不等于“已真机通过”。
- 凡涉及写操作、状态改变、支付、认证、消息发送、文件上传的步骤，均需要在后续审查中单独采集证据。

## 2. 注册和登录

- 起点：访客打开 `/login` 或进入受保护页面后被引导登录。
- 步骤：
  - 输入手机号与密码。
  - 调用登录接口建立会话。
  - 登录成功后返回对应页面或进入首页。
  - 刷新或重启客户端后尝试恢复会话。
- 页面：`/login`、登录后对应页面、`/profile`。
- 服务端状态变化：创建或恢复 JWT 会话，读取当前用户资料。
- 失败分支：手机号或密码错误、网络失败、服务未连接、会话过期。
- 取消分支：用户关闭登录页或返回上一个公开页面。
- 完成条件：用户进入已登录态，个人资料与底部导航状态正确。
- 用户下一步：浏览首页、进入“我的”、发布需求或物品。
- 当前是否完整：部分完整，代码与自动化具备基础，真机恢复与退出重登仍待复核。

## 3. 未登录浏览

- 起点：未登录访客打开首页、发现页、搜索页或物品/需求详情入口。
- 步骤：
  - 浏览公开内容。
  - 点击受保护入口时触发登录拦截。
  - 登录后返回原目标路径。
- 页面：`/`、`/discover`、`/search`、若干公开详情页。
- 服务端状态变化：公开查询，不写业务状态。
- 失败分支：公开接口失败、内容为空、跳转目标不存在。
- 取消分支：用户不登录，继续停留在公开浏览路径。
- 完成条件：未登录用户可以浏览，受保护动作不会绕过权限。
- 用户下一步：登录、继续浏览、进入搜索或详情。
- 当前是否完整：部分完整，公开浏览路径存在，登录拦截与对应页面回跳需要系统复核。

## 4. 首页到详情

- 起点：已登录或未登录用户进入首页。
- 步骤：
  - 查看快捷入口、推荐卡片、角色化模块。
  - 点击需求、工程师、旧物或回收入口。
  - 进入对应详情页。
- 页面：`/`、`/needs/[id]`、`/engineers/[userId]`、`/listings/[id]`、`/recycling/[id]`。
- 服务端状态变化：只读查询，可能写入轻量浏览统计或通知跳转留痕。
- 失败分支：列表为空、详情不存在、接口失败。
- 取消分支：用户返回首页或切换底部导航。
- 完成条件：首页可稳定加载并进入详情，详情信息完整且可返回。
- 用户下一步：发起联系、发起交易、发起需求或继续浏览。
- 当前是否完整：部分完整，结构已存在，首页真机巡检尚未完成。

## 5. 搜索到详情

- 起点：用户进入 `/search`。
- 步骤：
  - 点击输入框打开键盘。
  - 输入关键词或点击热门搜索。
  - 查看搜索结果、空态或失败态。
  - 点击某条结果进入详情。
- 页面：`/search`、相关详情页。
- 服务端状态变化：聚合检索查询，不应产生业务写入。
- 失败分支：查询失败、结果为空、结果与分类不匹配。
- 取消分支：第一次 Android 返回键收起键盘，第二次返回上一页。
- 完成条件：搜索输入、热门搜索、结果展示、返回行为和详情跳转均正确。
- 用户下一步：继续浏览详情、返回搜索结果、切换底部导航。
- 当前是否完整：部分完整；热门搜索点击后结果为空已有证据，详情跳转仍需复核。

## 6. 发布需求

- 起点：已登录用户从发布页、首页入口或“我的”进入创建需求。
- 步骤：
  - 进入多步骤向导。
  - 填写需求内容、预算、服务方式与补充信息。
  - 可保存草稿。
  - 最终发布需求。
- 页面：`/publish`、`/needs/create`、`/my-needs`、`/needs/[id]`。
- 服务端状态变化：创建 `needs` 草稿、更新草稿、发布为公开需求。
- 失败分支：校验失败、网络失败、AI 整理失败、发布失败。
- 取消分支：用户返回、放弃发布或保留草稿后退出。
- 完成条件：需求成功发布，并在“我的需求”与详情页可见。
- 用户下一步：等待方案、进入详情、查看评论与报价。
- 当前是否完整：部分完整；代码具备主流程，但未完成真机写操作审查。

## 7. 发布物品

- 起点：已登录用户从发布页进入物品创建。
- 步骤：
  - 填写标题、描述、价格、模式、成色等信息。
  - 选择并排序图片。
  - 提交发布。
- 页面：`/publish`、`/listings/create`、`/my-listings`、`/listings/[id]`。
- 服务端状态变化：创建 `items`、`listings`、图片文件记录。
- 失败分支：权限拒绝、图片上传失败、字段校验失败、网络失败。
- 取消分支：退出创建、保留未提交内容或返回发布页。
- 完成条件：新发布物品在我的发布和详情页可见。
- 用户下一步：编辑、下架、查看出价/申请、发起置换。
- 当前是否完整：部分完整；图片权限和上传需要真机写操作验证。

## 8. 编辑和管理

- 起点：用户进入“我的需求”“我的发布”“我的物品”。
- 步骤：
  - 查看列表。
  - 进入某一项详情或编辑页。
  - 修改可编辑字段，保存变更。
  - 在管理列表中确认回显。
- 页面：`/my-needs`、`/my-listings`、`/listings/[id]/edit`、`/my-items`、`/items/[id]`。
- 服务端状态变化：更新需求、更新发布、记录物品状态或历史。
- 失败分支：记录不存在、状态不允许编辑、保存失败。
- 取消分支：放弃修改并返回列表。
- 完成条件：修改保存后详情与列表一致，历史记录未丢失。
- 用户下一步：继续管理、查看详情、发起后续交易动作。
- 当前是否完整：部分完整；管理路径存在，但需要系统性写操作验证。

## 9. 置换完整流程

- 起点：用户从置换列表、物品详情或创建页进入置换。
- 步骤：
  - 选择目标物品与自己的可置换物品。
  - 发起置换请求。
  - 对方查看并接受、拒绝或取消。
  - 双方在详情中确认当前状态。
- 页面：`/swaps`、`/swaps/create`、`/swaps/[id]`、相关物品详情页。
- 服务端状态变化：创建 `swap_requests`，驱动物品与订单关联状态变化。
- 失败分支：物品不可置换、状态冲突、并发占用、网络失败。
- 取消分支：任一方在允许状态下取消。
- 完成条件：置换请求进入受控状态并可被双方查看。
- 用户下一步：继续确认、取消、查看关联订单或返回列表。
- 当前是否完整：部分完整；后端链路与测试存在，前端完整验收待补。

## 10. 回收完整流程

- 起点：用户发起回收询价，商家或回收方响应。
- 步骤：
  - 用户填写回收请求。
  - 商家查看并提交报价。
  - 用户比较报价并选择。
  - 流程进入订单承接。
- 页面：`/recycling/create`、`/recycling/[id]`、`/my-recycling`、商家相关列表。
- 服务端状态变化：创建 `recycling_requests`、`recycling_quotes`，在选择报价后生成订单承接关系。
- 失败分支：无报价、报价失效、订单创建失败、网络失败。
- 取消分支：用户取消询价或放弃选择。
- 完成条件：从询价进入报价和订单承接，状态展示一致。
- 用户下一步：查看订单、联系商家、取消或完成回收。
- 当前是否完整：部分完整；商家端与订单承接需要写操作验证。

## 11. 订单完整流程

- 起点：用户从项目、物品、回收、置换等业务线进入订单。
- 步骤：
  - 查看订单详情与当前状态。
  - 如需支付，进入支付页。
  - 履约过程中查看交付、验收、退款或投诉入口。
  - 完成后查看终态。
- 页面：`/orders`、`/orders/[id]`、`/payments/[orderId]`。
- 服务端状态变化：订单、支付、托管、退款、结算、投诉冻结等状态流转。
- 失败分支：支付失败、订单状态非法、退款被拒绝、投诉介入。
- 取消分支：在允许状态下取消或关闭。
- 完成条件：订单状态与后端一致，所有高风险动作都受后端约束。
- 用户下一步：评价、投诉、查看消息或返回列表。
- 当前是否完整：部分完整；后端能力完备，前端多业务线承接仍需审查。

## 12. 消息完整流程

- 起点：用户进入消息 Tab 或从详情页进入聊天。
- 步骤：
  - 查看会话列表与未读标识。
  - 进入某一会话。
  - 查看消息历史、回执状态。
  - 发送消息并回到列表。
- 页面：`/messages`、`/chat/[id]`。
- 服务端状态变化：读取 `conversations`、`messages`、`message_receipts`，发送时创建新消息与回执。
- 失败分支：会话不存在、消息发送失败、实时连接异常。
- 取消分支：用户退出会话并返回列表。
- 完成条件：列表、详情、未读、回执和实时更新一致。
- 用户下一步：继续聊天、查看订单或返回其他业务页面。
- 当前是否完整：部分完整；消息发送、回执、已读与发起会话均为 MUST_PASS，仍待受控写操作验证。

## 13. 通知跳转

- 起点：用户在消息 Tab 的通知页或独立通知页查看通知。
- 步骤：
  - 打开通知列表。
  - 查看通知内容与状态。
  - 点击通知跳转到目标业务页。
  - 返回通知列表。
- 页面：`/messages`、`/notifications`、目标详情页。
- 服务端状态变化：读取通知、标记已读、查询目标实体。
- 失败分支：目标实体不存在、通知重复、跳转失败。
- 取消分支：用户不点击通知，仅浏览列表。
- 完成条件：通知能跳转到正确页面，不产生错误跳转。
- 用户下一步：处理目标业务或继续浏览通知。
- 当前是否完整：部分完整；站内通知机制存在，跳转行为待系统复核。

## 14. 工程师身份

- 起点：普通用户进入认证中心并发起工程师认证。
- 步骤：
  - 查看工程师未认证状态。
  - 进入工程师申请页，填写资料并提交。
  - 审核通过后返回平台，获得工程师可用入口。
  - 进入需求、报价、项目相关流程。
- 页面：`/verifications`、`/engineer-apply`、`/engineers`、`/needs/[id]`、`/projects/[id]`。
- 服务端状态变化：创建并更新 `engineer_verifications`，审核后更新可接单资格。
- 失败分支：资料不完整、审核拒绝、补充材料、网络失败。
- 取消分支：用户退出申请，保留草稿或不再提交。
- 完成条件：认证状态、准入入口、报价与项目权限一致。
- 用户下一步：浏览需求、提交方案、进入项目履约。
- 当前是否完整：部分完整；工程师申请与材料提交属于 MUST_PASS，仍待受控写操作和真机路径复核。

## 15. 商家身份

- 起点：普通用户进入认证中心并发起商家认证。
- 步骤：
  - 查看商家未认证状态。
  - 进入商家认证页提交资料。
  - 审核通过后获取回收报价与承接能力。
- 页面：`/verifications`、`/merchant-apply`、`/recycling/[id]`。
- 服务端状态变化：创建并更新 `merchant_verifications`，审核后更新业务准入状态。
- 失败分支：资料不完整、审核拒绝、补充材料、网络失败。
- 取消分支：退出申请，暂不提交。
- 完成条件：认证通过后商家可以参与回收主流程。
- 用户下一步：查看询价、提交回收报价、进入订单承接。
- 当前是否完整：部分完整；代码已具备入口，真机与写操作验证不足。

## 16. 退出登录和重新登录

- 起点：已登录用户位于“我的”或设置页。
- 步骤：
  - 点击退出登录。
  - 会话与本地状态清理。
  - 用户再次进入登录页并重新登录。
  - 登录后恢复到正确用户状态。
- 页面：`/profile`、`/settings`、`/login`。
- 服务端状态变化：清理登录态 Cookie 或会话标识，重新创建新会话。
- 失败分支：退出失败、残留旧状态、重新登录后用户信息未刷新。
- 取消分支：用户放弃退出。
- 完成条件：退出后受保护页面重新要求登录，重新登录后进入正确账号。
- 用户下一步：继续正常使用。
- 当前是否完整：部分完整；自动化覆盖了登出行为，但真机写操作尚未执行。
