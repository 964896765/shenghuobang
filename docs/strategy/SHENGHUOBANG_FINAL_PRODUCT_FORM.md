# 生活帮最终产品形态

## 1. 正式产品定位

**生活帮——连接创意、生产、使用、流转和回收的产品全生命周期协作与资源交易平台。**

生活帮解决的不是“如何再做一个二手交易 App”，也不是一开始就建设复杂工业 ERP，而是解决同一个贯穿产品一生的问题：

> 让一个创意能够被实现，让一件物品能够被持续使用，让每一次生产、交易、维修、转手和回收都可协作、可验证、可追溯。

### 面向普通用户

买卖闲置、发布需求、找人解决问题，也可以把一个想法真正做出来。

### 面向企业和产业链

连接设计、工程、生产、销售、维修、回收和材料再利用的一站式协作平台。

### 品牌表达

- 主品牌：生活帮
- 产品副标题：让创意落地，让物品持续流转。
- 普通用户宣传：有需求，找生活帮；有闲置，上生活帮；有想法，也能在生活帮做出来。

### 品牌核心链路

想出来 → 做出来 → 买到手 → 用起来 → 修得好 → 转出去 → 回收掉 → 再利用

## 2. 最终三层产品结构

### 第一层：生活服务与资源流转

这是普通用户默认进入、理解成本最低、使用频率最高的一层：

- 发布生活需求
- 查找工程师、维修商和同城服务商
- 购买旧物、出售闲置、置换、赠送
- 回收估价、上门回收、交接结算
- 维修、翻新、租赁
- 订单、支付、托管、交付、售后和评价

**目标：** 承担用户增长、日常交易和生活服务，不向普通用户暴露 BOM、RFQ、生产批次等专业概念。

### 第二层：创意落地与新品孵化

面向创意者、设计师、工程师和小型创业团队：

- 发布创意，设置公开、私密或 NDA 可见范围
- 验证需求，收集意向和专业评审
- 邀请设计师、工程师和项目负责人
- 形成项目、里程碑、设计版本和原型交付
- 样机测试、成本评估、意向登记和预售筹措
- 对接供应商与生产商，完成试产、质检和新品发布

**目标：** 让“奇思妙想”从内容展示变成可执行、可交付、可验收的产品项目。

### 第三层：产品生命周期与产业链追踪

面向品牌方、生产商、供应商、维修商、回收商和企业客户：

- 产品型号、设计版本和 BOM
- 供应商、生产订单、生产批次和质检
- 单件产品身份和数字档案
- 所有权、质保、维修、部件更换、折损和翻新
- 回收、拆解、零件再利用、材料再生和去向证明
- 企业资产处置、批量维修、批量回收和 ESG 报告

**目标：** 建立生活帮长期竞争壁垒，使新产品、旧物交易、维修和回收共享同一条产品事实链。

### 横向核心：可信内容生态

内容创作不是附属信息流，而是连接三层产品结构的传播、经验沉淀和业务转化中枢。内容创作、发现分发、统一发布入口和创作者中心共享同一内容事实源；内容可关联需求、创意、筹措、产品、单件、商品和服务，并以来源标签、身份认证、AI 标识、字段脱敏和审核记录进入高可信体系。

详细边界见 [内容创作、发现分发与创作者生态蓝图](../architecture/CONTENT_CREATION_DISCOVERY_ECOSYSTEM_BLUEPRINT.md)。

## 3. 最终参与者体系

角色不直接固化在账号上。平台以“账号 + 多业务身份 + 认证资质 + 组织成员关系 + 项目角色 + 平台职务”表达参与者。

### 普通个人用户

默认身份。核心入口：发布需求、附近旧物、维修、回收估价、置换赠送、订单和个人物品。

### 创意发起人

提交创意、设置保密范围、招募协作者、形成项目、验收交付、开放意向登记并跟踪后续生产和流转。

### 设计师与工程师

展示技能和作品，报价、参与项目、提交设计版本和交付物、完成里程碑、处理返工、获得验收和阶段款。

### 商家与服务商

二手商家、维修店、翻新商、租赁商、鉴定机构、估价师、物流与仓储服务商。以组织或个人工作室方式经营。

### 生产和供应链企业

原材料商、零部件商、代工厂、组装厂、包装商、检测机构、品牌方和经销商。通过授权资料、RFQ、BOM、生产批次和质检协作。

### 回收产业链

上门回收员、回收商、分类处理商、拆解企业、废品商、再制造企业和再生材料企业。记录回收交接、拆解结果、零件和材料去向。

### 企业客户

企业闲置资产处置、批量采购、维修、回收、员工设备管理、资产生命周期和循环利用报告。

### 平台运营人员

客服、内容审核、认证初审、认证复审、投诉调查、投诉裁定、财务审核、资金执行、安全审计、权限管理员和超级管理员。平台职务与业务身份完全分离并实行职责分离。

## 4. App 最终交互形态

### 统一账号、多身份、多工作台

- 所有账号默认具有普通用户身份。
- 用户可以申请多个业务身份，并分别提交认证。
- 用户可加入多个组织，在“我的身份/工作台”中切换当前工作上下文。
- 权限由服务端根据身份、组织、项目、资源、状态和保密级别共同判定，客户端工作台仅决定入口展示，不决定真实权限。

### 普通用户底部导航

1. 首页
2. 发现
3. 发布
4. 消息
5. 我的

### 首页

- 当前城市与搜索
- 发布需求
- 附近服务
- 附近旧物
- 回收估价
- 维修服务
- 置换赠送
- 创意专区
- 我的进行中事项

### 发现

顶部内容频道：推荐、关注、产品、创意、经验、视频、问答、附近；结构化需求、工程师、旧物、回收、新品和服务继续通过搜索、频道关联卡片和业务入口访问。

### 发布

根据当前身份分成“业务发布”和“内容创作”。业务发布包括需求、旧物、回收、创意、新品、服务和生产询价；内容创作包括图文、视频、文章、问答、产品测评、教程、创意/筹措进展和维修案例。

### 消息

聊天、报价、项目通知、订单通知、系统通知和审核通知统一聚合，但按类型分组。

### 我的

订单、发布、项目、物品、创意、产品档案、创作者中心、身份认证、组织、资金退款、设置隐私和工作台切换。

## 5. 专业工作台最终形态

专业用户不需要安装另一个 App；工作台根据身份、组织成员关系和能力动态出现。

### 设计师/工程师工作台

待报价、参与项目、任务与里程碑、设计版本、文件交付、作品集、收益和认证。

### 商家/维修工作台

商品、服务、库存、订单、客户、售后、员工和经营数据。

### 生产商工作台

询价、报价、授权资料、生产订单、BOM、工程变更、批次、质检、产品实例和交付。

### 回收商工作台

附近回收需求、报价、上门任务、复检、交接、拆解去向、结算和回收证明。

### 企业工作台

组织成员、资产、采购、项目、维修、处置、回收、数据权限和报告。

## 6. 三个核心闭环

### 交易闭环

发现 → 沟通 → 报价 → 下单 → 托管 → 交付 → 验收 → 结算 → 售后

### 产品闭环

创意 → 设计 → 原型 → 生产 → 销售 → 使用 → 维修 → 易手 → 翻新 → 回收

### 材料闭环

材料 → 零部件 → 产品 → 使用 → 拆解 → 再生材料 → 新产品

## 7. 产品数字档案

高价值或选择追踪的物品拥有单件数字档案。对普通用户展示：

- 产品身份、型号和生产批次
- 可公开质检结果、当前成色和质保状态
- 经授权的维修与翻新记录
- 易手次数、当前估值和回收建议

内部完整档案还包括设计版本、材料批次、供应商、生产商、检测机构、所有权变化、部件更换、折损、回收、拆解和材料再利用。不同身份只看到业务必要的数据，历史所有人的联系方式永不随物品转移。

## 8. 复杂度控制原则

- 普通用户首页永不直接展示 BOM、RFQ、生产批次、材料追溯、财务审核和审计事件。
- 专业能力只在认证后或加入组织后显示。
- 每个用户界面只呈现当前任务需要的信息。
- 后台复杂性通过工作台和权限上下文隔离，而不是堆叠在同一页面。
- 新模块必须复用统一账号、身份、组织、项目、物品、订单、资金、文件、消息、审计和状态机。

## 9. 最终形态与版本主线

1. **V3.3-A：身份、组织与权限底座**
2. **V3.3-B：创意落地最小闭环**
3. **V4.1：生产协作**
4. **V4.2：产品数字档案与流转**
5. **V4.3：回收与材料闭环**

云平台、金额、支付、安全、AI、位置、搜索和发布质量作为跨阶段技术轨提供支撑，不替代上述产品主线。
