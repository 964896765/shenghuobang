# 生活帮最终 App 信息架构与工作台形态

## 1. 设计目标

生活帮最终只保留一个 App、一个账号体系和一套消息/订单/文件/审计基础设施，但根据用户当前身份、组织和任务动态展示不同工作台。

平台复杂度存在于服务端领域模型和专业工作台中，不直接堆到普通用户首页。客户端必须遵循：

- 默认进入个人模式；
- 普通用户只看到生活需求、旧物、维修、回收和订单；
- 认证身份或组织成员关系成立后，才出现专业工作台；
- 工作台切换只改变入口、默认筛选和当前上下文，不改变服务端真实权限；
- 同一能力只能有一个领域事实源，不能为每个工作台复制一套订单、消息、文件或资金系统。

## 2. 最终全局导航

普通用户和专业用户都使用同一套五 Tab：

1. 首页
2. 发现
3. 发布
4. 消息
5. 我的

专业功能不增加第六个固定 Tab，而是在“我的 → 切换工作台”后改变首页内容、发布入口、消息筛选和待办卡片。

## 3. 个人模式最终形态

### 3.1 首页

首屏只展示普通用户可理解的任务：

- 当前城市与搜索；
- 发布需求；
- 附近服务；
- 附近旧物；
- 回收估价；
- 维修服务；
- 置换与赠送；
- 创意专区；
- 我的进行中事项；
- 最近浏览与推荐。

首页不得直接展示：BOM、RFQ、生产批次、材料批次、质检工单、资金审核、审计日志。

### 3.2 发现

顶部分类：

- 需求；
- 工程师；
- 旧物；
- 回收；
- 新品；
- 创意；
- 服务。

所有搜索结果先经过服务端权限过滤。私密创意、内部组织资料、未授权报价和保密附件不得通过搜索摘要泄露。

### 3.3 发布

个人模式至少提供：

- 发布需求；
- 发布旧物；
- 发起回收；
- 发布创意；
- 发布服务（仅具备有效专业身份时）；
- 发布新品（仅具备对应能力时）；
- 创建生产询价（仅组织工作台可见）。

### 3.4 消息

统一消息中心按业务类型分组：

- 聊天；
- 报价；
- 项目通知；
- 订单通知；
- 审核通知；
- 系统通知。

会话可以关联需求、创意、项目、订单、组织或回收任务，但消息正文和附件访问仍由资源权限控制。

### 3.5 我的

个人模式下展示：

- 我的订单；
- 我的发布；
- 我的项目；
- 我的物品；
- 我的创意；
- 产品档案；
- 身份认证；
- 我的组织；
- 资金与退款；
- 工作台切换；
- 设置与隐私。

## 4. 工作台选择器

工作台列表由服务端根据有效身份、组织成员关系、项目成员关系和平台职务生成。

### 4.1 可出现的工作台类型

- 个人模式；
- 设计师工作台；
- 工程师工作台；
- 商家工作台；
- 维修工作台；
- 生产商工作台；
- 供应商工作台；
- 质检机构工作台；
- 回收商工作台；
- 企业工作台；
- 平台运营工作台。

### 4.2 工作台卡片字段

- 工作台名称；
- 个人身份或组织名称；
- 当前岗位；
- 认证状态；
- 待办数量；
- 是否存在权限受限或认证过期；
- 最近使用时间。

### 4.3 切换规则

- 切换个人/组织工作台不刷新登录态；
- 每个请求仍携带账号会话，工作台上下文只作为授权条件之一；
- 无效、停职、已退出或认证过期的工作台不可进入；
- 权限被回收后，已打开页面下一次请求必须立即失败并刷新入口；
- 客户端不得通过修改本地 workspaceId 获得额外数据。

## 5. 专业工作台最终形态

- 待提交设计版本；
- 待处理返工；
- 待验收协作项；
- 收益和结算状态。

核心模块：

- 报价；
- 项目；
- 设计任务；
- 设计版本；
- 文件交付；
- 作品集；
- 认证；

## 6. 五个主阶段结束后的 App 实际形态

### V3.3-A 完成后

普通用户首页和现有交易流程基本不变；“我的”中新增身份、认证、组织和工作台切换。项目支持多成员，专业用户看到最小工作台壳层。尚未出现完整创意、生产或回收产业链页面。

### V3.3-B 完成后

首页和发现新增“创意专区”；发布入口新增发布创意；设计师/工程师工作台出现邀请、项目、设计版本、原型里程碑和验收；普通用户可登记意向，但不会被误导为已经付款或众筹。

### V4.1 完成后

成熟创意项目可进入生产协作；生产商、供应商和质检机构工作台正式开放；普通用户只在项目进度和新品页面看到可理解的生产摘要，不看到内部报价、BOM 和批次细节。

### V4.2 完成后

“我的物品”和商品详情升级为产品数字档案入口；新品和旧物共享所有权、维修、成色、估值和流转历史；扫码可进入授权层级对应的档案视图。

### V4.3 完成后

回收入口与产品档案连接；用户可看到回收估价、交接、结算和去向证明；回收组织可以管理复检、拆解、零件和再生材料批次；材料来源可追溯到产品实例。

## 7. 页面复杂度控制门禁

每次新增页面都必须回答：

1. 这是个人模式还是哪个专业工作台？
2. 用户当前要完成的唯一主要任务是什么？
3. 页面是否暴露了与当前任务无关的专业字段？
4. 是否复用了统一订单、文件、消息、资金和审计？
5. 无权限时是隐藏入口、展示脱敏摘要，还是明确拒绝？
6. 身份停职、组织退出、项目移除后页面如何立即失效？

不能回答以上问题的页面不得进入开发。
