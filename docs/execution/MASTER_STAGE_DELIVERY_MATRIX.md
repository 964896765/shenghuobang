# 生活帮主阶段交付总矩阵

## 1. 唯一产品主路线

```text
V3.3-A 身份与组织权限底座
  ↓
V3.3-B 创意落地最小闭环
  ↓
V4.1 生产协作
  ↓
V4.2 产品数字档案与流转
  ↓
V4.3 回收与材料闭环
```

各阶段只在前一阶段门禁通过后开始。金额、云环境、支付、安全、AI、位置和发布质量作为跨阶段技术轨穿插执行，不得取代主路线。

## 2. V3.3-A：多身份、组织与数据权限基础工程

### A1 规格与权限威胁模型

**目标：** 冻结目标模型和迁移/授权规则，不改业务代码。

**产物：**

- 新增表数据字典；
- 旧字段迁移矩阵；
- 身份、认证、组织、成员、邀请、项目成员和平台职务状态机；
- 第一版能力目录；
- RBAC + ABAC + Data Scope + Field Mask 决策表；
- 字段脱敏矩阵；
- 职责分离矩阵；
- 第一批路由改造清单；
- 越权、并发、迁移和回滚测试矩阵。

**停止条件：** 文档链接、规范校验和差异检查通过；不得自动进入 A2。

### A2 追加数据库迁移与兼容回填

**新增表：**

- business identities / identity profiles；
- qualification types / qualifications / documents / actions；
- organizations / memberships / invitations；
- organization roles / capabilities / role capabilities / member roles；
- project members / project member roles；
- platform staff assignments；
- workspace preferences；
- authorization audit logs。

**回填：**

- 每个账号生成 consumer 身份；
- 工程师、商家状态迁入身份和认证；
- 旧后台角色迁入平台职务；
- owner/engineer 回填项目成员；
- currentRole 回填工作台偏好。

**门禁：**

- 空库迁移；
- V3.2.4 升级；
- 重复执行不重复数据；
- 冲突记录可导出；
- 恢复备份后可再次执行。

### A3 授权内核与兼容适配

**服务：**

- AuthorizationService；
- WorkspaceContextService；
- SensitiveDataMaskService；
- legacy role/verification/project adapters。

**优先改造：**

- 项目详情；
- 项目文件上传、签名、禁用和下载日志；
- 里程碑提交与验收；
- 后台高风险接口；
- 认证读取；
- 消息会话资源授权。

**门禁：** 默认拒绝、稳定 reasonCode、服务端数据范围过滤、工作台伪造无效。

### A4 身份、认证和组织服务

**API：**

- 我的身份、创建/停用身份；
- 认证申请、补件、撤回和状态；
- 创建组织、组织详情、我的组织；
- 邀请、接受、拒绝、撤销、过期；
- 成员停职、恢复、移除和退出；
- 组织角色和能力分配；
- 权限变更审计。

**门禁：** 邀请并发幂等、最后所有者保护、跨组织隔离、成员停职即时失效。

### A5 项目成员、平台职务和最小 UI

**API：**

- 项目成员邀请/接受/移除；
- 项目角色分配；
- 平台职务分配/停职/撤销；
- 我的可用工作台；
- 当前工作台切换。

**页面：**

- 我的身份；
- 身份认证状态；
- 我的组织；
- 组织成员与邀请；
- 工作台切换；
- 项目成员；
- 最小专业工作台壳层。

**门禁：** 普通用户原首页不变；多身份不复制账号；项目只读成员不能写；提交者不能自验收。

### A6 安全、越权和迁移验证

**测试至少覆盖：**

- 同账号多身份；
- 同账号多个组织；
- 跨组织直接 ID；
- 停职、退出、移除后的 API、WebSocket 和文件签名；
- 项目非成员、只读成员、已移除成员；
- 后台初审/复审、调查/裁定、审核/执行分离；
- 并发邀请、重复接受、过期邀请；
- 旧账号和旧项目回归；
- 迁移失败重跑与恢复。

### A7 真机验收和阶段交付

**真机流程：**

- 普通用户登录和现有旧物/需求流程；
- 创建第二身份并查看认证状态；
- 创建组织并邀请成员；
- 切换个人与组织工作台；
- 成员停职后权限回收；
- 项目邀请与只读查看；
- 后台职责分离演示。

**交付：** 报告、未决问题、数据库备份说明、干净源码包。

## 3. V3.3-B：创意落地最小闭环

### B1 创意与保密规格

- idea 状态机；
- public/private/nda 可见范围；
- NDA 签署、撤销和过期；
- 创意到项目幂等转换；
- 设计版本和原型交付规则；
- 意向登记不是付款或众筹承诺；
- 搜索、通知和日志泄露威胁模型。

### B2 数据迁移

**新增：** ideas、visibility grants、collaboration invitations、NDA agreements、interest registrations、design version metadata。

**复用：** projects、milestones、requirements、changes、files、acceptances、messages、notifications。

### B3 服务端闭环

- 创意草稿、发布、编辑和归档；
- 可见范围；
- 邀请设计师/工程师；
- NDA；
- 创意转项目；
- 设计版本；
- 原型里程碑；
- 提交、返工和验收；
- 意向登记和匿名统计。

### B4 客户端

- 创意专区；
- 创意创建/详情；
- 保密设置；
- 协作者邀请；
- 项目成员；
- 设计版本；
- 原型里程碑；
- 交付验收；
- 意向登记；
- 我的创意工作台。

### B5 协作和文件安全

- 私密创意不出现在未授权搜索/推送；
- NDA 撤权后文件签名失效；
- 设计版本追加，不覆盖；
- 返工形成新提交；
- 提交者不能验收；
- 访问审计可查询。

### B6 沙箱资金和弱网验证

仅在需要阶段款时启用沙箱，不开放真实资金。验证离线重试、重复提交、文件断点和消息顺序。

### B7 真机和交付

完整走通：发布创意 → 设置 NDA → 邀请 → 接受 → 转项目 → 设计版本 → 原型提交 → 验收 → 意向登记。

## 4. V4.1：生产协作

### P1 生产领域规格

- RFQ、报价版本和报价隔离；
- 设计发布；
- BOM 版本；
- 生产订单；
- 试产/量产批次；
- 检验计划和结果；
- 不合格处理；
- 工程变更；
- 产品实例生成。

### P2 组织身份扩展与认证

增加 supplier、manufacturer、inspection provider、brand owner 身份和组织类型；扩展生产/检测认证，不改账号模型。

### P3 RFQ 和报价

- 邀请组织；
- 授权资料包；
- 独立报价；
- 报价版本；
- 比较和选定；
- 竞争报价隔离；
- 过期和撤回。

### P4 设计发布、BOM 和工程变更

- 草稿设计与发布设计分离；
- BOM 版本与设计发布绑定；
- 生产商只能提出工程变更；
- 已生产批次不被新版本覆盖。

### P5 生产订单、批次和质检

- 试产/量产订单；
- 材料和工艺确认；
- 批次记录；
- 质检、不合格、返工、报废；
- 检测报告；
- 出厂状态。

### P6 产品实例和集成验证

- 唯一序列策略；
- 幂等生成；
- 型号/批次/质检关联；
- 订单、文件、消息和资金集成；
- 竞争组织越权测试。

### P7 真机和交付

生产商、供应商、质检机构各自工作台走通一条试产链，并确认普通用户只看到可理解摘要。

## 5. V4.2：产品数字档案与流转

### L1 产品档案规格

- product model / model version；
- product instance；
- 公开、所有者、服务商、生产组织和平台审计视图；
- 所有权、质保、维修、成色、折损、估值和翻新状态机；
- 历史所有者隐私。

### L2 数据模型和旧物适配

- 新品实例从生产批次生成；
- 现有 item 可选补建档案；
- listing 关联 item/product instance；
- 现有 ownership/service/status history 迁移适配。

### L3 所有权和质保

- 首次激活；
- 订单交付触发转移；
- 当面交接验证；
- 争议锁；
- 质保权益和售后责任。

### L4 维修、部件和翻新

- 检测、报价、维修、测试、交付；
- 部件实例和更换；
- 质保期；
- 翻新批次和证明；
- 维修商只看必要记录。

### L5 成色、折损和估值

- 评估输入、证据和规则版本；
- 估值快照；
- 人工/机构复核；
- 不允许覆盖旧评估；
- AI 只给建议，不作为最终认证。

### L6 流转集成

出售、赠送、置换、租赁和回收入口与产品档案连接；所有权与订单状态保持一致。

### L7 真机和交付

从新品/旧物建档到维修、再次出售和所有权转移，验证公开视图和隐私视图。

## 6. V4.3：回收与材料闭环

### R1 回收和材料规格

- 回收申请、初估、报价、复检、最终报价、交接、结算；
- 拆解批次；
- 零件分流；
- 材料分类、重量、纯度和损耗；
- 再生材料批次和去向；
- 回收证明。

### R2 现有回收能力升级

复用 `recycling_requests/recycling_quotes`，新增上门/邮寄、复检、报价版本、交接和所有权终止/转移。

### R3 回收履约和结算

- 回收任务；
- 回收员分配；
- 位置最小披露；
- 最终价格确认；
- 争议；
- 结算和凭证。

### R4 拆解和零件

- 拆解投入；
- 产出零件；
- 可复用/维修/报废分类；
- 来源产品实例；
- 质量和库存状态。

### R5 再生材料

- 材料类型；
- 重量和纯度；
- 检测证据；
- 批次；
- 损耗核对；
- 组织间转移；
- 新产品领用关联。

### R6 隐私、审计和闭环验证

- 回收员只看任务必要地址；
- 回收商看不到历史所有者隐私和无关交易价；
- 去向变更追加更正；
- 投入产出质量差异可解释；
- 材料可反向追踪产品实例。

### R7 真机和交付

走通用户回收、回收商复检交接、拆解企业记录、再生材料批次和新产品来源关联。

## 7. 跨阶段硬门禁

| 技术轨 | 触发点 | 必须完成的最低结果 |
|---|---|---|
| 金额 | 阶段款、生产采购、回收结算 | 整数分、对账、幂等、双写/切换策略 |
| 云环境 | 多用户外部协作前 | 隔离数据库、HTTPS/WSS、对象存储、Push、日志和备份 |
| 支付 | 真实资金前 | Provider 沙箱、回调验签、托管、退款、结算、职责分离 |
| 安全隐私 | 每阶段 | 默认拒绝、字段脱敏、撤权、审计、密钥管理 |
| AI/位置 | 对应流程 | AI 不做最终授权；位置按需且最小精度 |
| 发布质量 | 每阶段 | 迁移、单元、集成、越权、真机、弱网、备份恢复 |

## 8. 阶段报告统一格式

每个批次报告必须包含：

- 实际修改文件；
- 实际新增表/API/页面；
- 自动化验证结果；
- 未验证环境；
- 兼容字段；
- 未完成项；
- 风险和回滚；
- 是否允许进入下一批。

不得把“已规划”“已写 Stub”“本地静态通过”写成“完整闭环已完成”。
