# 生活帮系统开发总规划：从身份底座到材料闭环

## 1. 规划原则

后续开发采用“先横向底座，再纵向闭环”的方式：

1. 先建立可扩展的身份、组织和权限骨架。
2. 立即用创意落地 MVP 验证骨架，而不是继续空建角色系统。
3. 在已验证的成员、项目、文件和权限模型上增加生产协作。
4. 把新品、旧物、维修、易手和翻新接入统一产品数字档案。
5. 最后延伸到回收、拆解、零件和再生材料闭环。

金额、云服务、支付、安全和发布质量属于跨阶段技术轨，只作为进入相关业务的硬门禁，不再取代产品主路线。

## 3. 五条产品主线

| 顺序 | 版本 | 最终增量 | 验证的核心问题 |
|---|---|---|---|
| 1 | V3.3-A | 多身份、认证、组织、项目成员、能力权限和工作台 | 同一账号在不同组织和项目中能否安全协作 |
| 2 | V3.3-B | 创意发布到原型验收和意向登记 | 权限底座能否承载真实纵向闭环 |
| 3 | V4.1 | RFQ、BOM、生产订单、批次、质检和产品实例 | 多组织和保密资料能否协同生产 |
| 4 | V4.2 | 产品型号、单件身份、所有权、维修、折损、易手和翻新 | 新品和旧物能否共享一条产品事实链 |
| 5 | V4.3 | 回收、复检、拆解、零件再利用、再生材料和去向 | 产品链能否闭合为材料循环链 |

## 4. V3.3-A：多身份、组织与数据权限基础工程

### 完成后平台应呈现

- 普通用户仍按原路径使用生活需求、旧物、回收和订单。
- “我的”新增身份与工作台入口。
- 一个账号可以同时拥有普通用户、设计师、工程师等身份。
- 认证状态独立于身份状态。
- 用户可以创建/加入组织，并在多个组织间切换工作台。
- 项目支持多成员和项目角色，不再只认 owner/engineer 两人。
- 所有高风险接口通过统一授权服务判定。
- 组织成员停职、退出或被移除后权限立即回收。
- 后台职务从普通业务身份中分离。

### 不做

不开发完整生产、回收和供应链页面；不一次性展示所有身份；不删除旧角色字段；不接真实支付。

任务书：[V3.3-A 身份组织权限基础工程](./stages/V3_3_A_IDENTITY_ORG_PERMISSION_FOUNDATION.md)

## 5. V3.3-B：创意落地最小闭环

### 完成后平台应呈现

创意发起人可以发布公开/私密/NDA 创意，邀请设计师和工程师，形成多成员项目，管理设计版本和原型里程碑，提交交付、完成验收并开放意向登记。

### 最小参与身份

- 创意发起人
- 设计师
- 工程师
- 项目负责人
- 受邀查看者

### 核心复用

需求、报价、项目、里程碑、项目需求、文件交付、消息、通知、认证、投诉和信用。

### 不做

不进入完整 RFQ、BOM、量产和正式预售资金；生产商和回收商工作台不在本阶段开放。

任务书：[V3.3-B 创意落地最小闭环](./stages/V3_3_B_IDEA_TO_PROTOTYPE_MVP.md)

## 6. V4.1：生产协作

### 完成后平台应呈现

被授权的供应商、生产商和检测机构可以围绕已成熟的创意项目完成 RFQ、报价比较、设计发布、BOM、生产订单、试产、工程变更、生产批次、质检和产品实例编号。

### 权限重点

- 供应商只能看到自己被邀请的 RFQ 和授权附件。
- 不允许供应商之间互看报价。
- 生产商不能修改已发布设计版本，只能发起工程变更。
- 质检方只能提交被指派批次的检验结果。
- 生产组织员工只看到其组织和被授权项目的数据。

任务书：[V4.1 生产协作](./stages/V4_1_PRODUCTION_COLLABORATION.md)

## 7. V4.2：产品数字档案与流转

### 完成后平台应呈现

新品生产产生产品型号和单件实例；现有旧物可补建档案。所有权、质保、维修、部件更换、成色、折损、估值、易手和翻新进入同一产品数字档案，并与发布、订单和售后连接。

### 用户侧表现

普通用户扫码或从订单进入，只看到公开和授权摘要；专业组织在权限范围内查看生产、质检和服务记录；历史所有者个人信息不随产品转移。

任务书：[V4.2 产品数字档案与流转](./stages/V4_2_PRODUCT_PASSPORT_AND_CIRCULATION.md)

## 8. V4.3：回收与材料闭环

### 完成后平台应呈现

现有回收申请扩展为：初估、报价、上门/邮寄、实物复检、最终价格、交接、回收订单、拆解、可复用零件、材料称重纯度、再生材料批次、去向证明和新产品材料关联。

### 隐私重点

回收参与者只看到履约必要信息，不得到历史所有者联系方式、非必要交易价格和无关维修隐私。

任务书：[V4.3 回收与材料闭环](./stages/V4_3_RECYCLING_AND_MATERIAL_LOOP.md)

## 9. 跨阶段技术轨

| 技术轨 | 当前状态 | 必须在哪个主阶段前完成 |
|---|---|---|
| T1 金额统一与对账 | Phase 1 已完成 | V3.3-B 启用阶段款前完成双写；V4.1 生产资金前完成切换 |
| T2 云环境、对象存储、Push、可观测性 | 部分本地能力 | V3.3-B 外部协作测试前具备稳定测试环境 |
| T3 支付、退款、托管、结算 | 沙箱和表结构基础 | V3.3-B 只可沙箱；正式资金在专项门禁通过后开放 |
| T4 隐私、安全、密钥、合规 | 已有基础审计 | V3.3-A 完成授权和脱敏；每阶段持续加强 |
| T5 AI、位置、搜索和推荐 | 需求结构化和位置基础 | 按具体用户流程接入，不作为权限来源 |
| T6 测试、迁移、灾备和发布 | V3.2.4 有真机基线 | 每阶段必须通过自己的发布门禁 |

详见 [跨阶段技术轨](./tracks/CROSS_CUTTING_TECHNICAL_TRACKS.md)。

## 10. 阶段进入条件

### V3.3-A → V3.3-B

- 旧用户、工程师、商家和后台账号迁移结果可核对。
- 组织成员、项目成员、邀请、停职和退出状态机通过。
- 关键接口已接入授权服务，越权测试通过。
- 工作台切换不改变服务端真实权限。

### V3.3-B → V4.1

- 私密/NDA 创意无越权读取。
- 多成员项目、设计版本、原型里程碑和验收闭环稳定。
- 交付提交者不能自验收。
- 项目文件访问、撤权和签名失效通过。

### V4.1 → V4.2

- RFQ、报价隔离、设计发布、BOM、生产批次和质检可追溯。
- 工程变更不覆盖历史设计和 BOM。
- 产品实例编号生成具有幂等性和唯一性。

### V4.2 → V4.3

- 产品型号、实例、所有权、维修和状态历史稳定。
- 历史所有者隐私隔离通过。
- 估值、折损和翻新记录是追加式历史，不静默覆盖。

## 11. 每个阶段统一执行批次

1. 规格与威胁模型
2. 追加数据库迁移与回填
3. 服务端领域服务、权限、事务和审计
4. 客户端入口、工作台和流程页面
5. 自动化测试、越权测试和迁移测试
6. MySQL、真机、弱网和外部服务沙箱验证
7. 阶段报告、源码归档和下一阶段门禁

Codex 每次只接收一个批次任务。批次未通过不得自动进入下一批，也不得顺手开发后续模块。
