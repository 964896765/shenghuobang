# 跨阶段技术轨

这些工作不再作为产品主路线版本，而是在需要时作为硬门禁穿插执行。

## T1 金额与财务正确性

- 保留已完成的 30 个金额字段注册表和整数分契约。
- 完成影子列、回填、双写、读切换、旧列退役和逐表对账。
- 创意项目阶段款、生产订单和回收结算不得使用浮点数。
- 正式资金开放前必须通过日终对账、退款、托管和结算一致性测试。

## T2 云环境与外部服务

- 隔离测试数据库、HTTPS/WSS、对象存储、Push、日志指标、备份和恢复。
- 项目文件、NDA、图纸、质检报告必须通过存储 Provider 和短签名访问。
- 外部服务使用 Provider 接口，业务层不得绑定供应商 SDK。

## T3 支付与资金

- 先沙箱，后正式 Provider。
- 所有支付回调验签、幂等和金额匹配由服务端处理。
- 阶段款、订单款、退款、托管、结算和出款使用同一账务体系。
- 财务审核和资金执行职责分离。

## T4 安全、隐私与合规

- 默认拒绝授权、字段脱敏、敏感访问目的记录、权限变更审计。
- NDA、组织资料、报价、图纸、生产数据和产品档案分级。
- 账号注销、数据导出、认证撤销、成员离职和文件撤权具备完整流程。
- 密钥不进入源码包；定期轮换并记录。

## T5 AI、位置、搜索与推荐

- AI 用于结构化需求、识别物品、估价辅助和流程建议，不直接批准权限、付款、认证或质检。
- 位置继续采用前台按需、约 1km 精度、可手动选择城市的原则。
- 搜索统一覆盖需求、服务、旧物、创意、新品和组织，但结果按权限过滤。

## T6 质量、迁移与发布

- 每个阶段均要求空库迁移、指定历史版本升级、失败重跑和恢复演练。
- 高风险接口必须有越权、并发、幂等和审计测试。
- 正式发布前完成真机矩阵、弱网、性能、备份恢复、隐私文案和唯一 RC 包。
