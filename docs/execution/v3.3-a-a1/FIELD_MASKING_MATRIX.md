# V3.3-A / A1 敏感字段矩阵

## 1. 返回等级

```text
OMIT      字段不存在于响应结构
MASK      只返回脱敏显示值
SUMMARY   返回业务必要摘要，不返回原值
FULL      返回最小必要原值，必须有能力、目的、短时授权和审计
```

服务端完成字段裁剪后再序列化；客户端隐藏不算脱敏。日志、埋点、错误、通知、搜索索引和审计 `contextData` 一律不得含原值。高敏文件只返回一次性短签名，不返回 storageKey/永久 URL。

## 3. 适用于所有字段的强制规则

1. 列表接口永远比详情接口少；不能因前端“不展示”而返回高敏字段。
2. `FULL` 必须同时满足 capability、data scope、资源关系、业务状态、保密/NDA、purpose 和职责分离。
3. 通知/Push 只发最小摘要；私密资源不得泄露标题、文件名、报价、参与者或拒绝细节。
4. 导出是独立高风险动作；普通“查看”能力不自动包含导出。
5. 权限撤销后新请求立即失败；已签发链接在内容端点实时复核；客户端收到撤权事件后清缓存。
6. `permission_audit_events.fieldMask` 只记录规则名和被裁剪字段名，不记录原值。
7. 当前代码中工程师/商家公开详情、认证文件、通用文件和项目文件都必须在 A3 对照本矩阵整改；A1 不改 API。
