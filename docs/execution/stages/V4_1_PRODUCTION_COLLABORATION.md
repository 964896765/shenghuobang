# V4.1 生产协作

## 1. 用户可见结果

成熟创意项目可以邀请供应商、生产商和检测机构，发送 RFQ，比较报价，发布设计和 BOM，建立试产/量产订单，记录批次质检，并为合格产品生成单件实例编号。

## 2. 新身份和组织类型

supplier、manufacturer、inspection_provider、brand_owner。它们通过 V3.3-A 身份/组织目录和认证体系扩展，不修改账号模型。

## 3. 流程

设计冻结 → RFQ → 邀请组织 → 授权资料 → 独立报价 → 选定供应商/生产商 → 生产协议 → 设计发布 → BOM → 生产订单 → 材料确认 → 试产 → 检测 → 工程变更 → 量产 → 批次质检 → 出厂 → 产品实例

## 4. 领域数据

- RFQ、RFQ 邀请、报价和报价版本
- 设计发布包和授权范围
- BOM、BOM 版本和 BOM 行
- 生产订单、工序和材料确认
- 试产/量产批次
- 检验计划、检验结果和不合格项
- 工程变更请求/批准
- 产品实例编号生成批次

## 5. 数据隔离

- 各报价组织只看自己的报价。
- 设计师只能看与设计任务相关的成本摘要，不自动看到供应商底价。
- 生产商只能读取已发布设计包，不能覆盖设计源版本。
- 检测机构只看被指派批次、检验标准和必要产品信息。
- 生产组织成员受组织岗位和订单分配限制。

## 6. 状态机重点

RFQ、生产报价、设计发布、BOM、生产订单、批次、检验、不合格处理、工程变更和产品实例均采用追加历史和明确状态迁移。

## 7. 验收门禁

- 竞争报价隔离和附件授权不可绕过。
- BOM 版本和设计发布版本可追溯。
- 工程变更不会静默修改已生产批次。
- 批次质检和不合格品处理闭环。
- 产品实例编号唯一、幂等并关联型号/批次。
