# V4.2 产品数字档案与流转

## 1. 用户可见结果

新品生产后自动建立产品型号和单件档案；旧物可由用户补录或认证机构建档。交易、维修、翻新、质保、成色、折损、估值和所有权变化形成可追踪历史。

## 2. 流程

产品型号 → 单件实例 → 首次销售 → 所有权激活 → 使用 → 维修/部件更换 → 成色评估 → 易手/赠送/置换/租赁 → 翻新 → 回收建议

## 3. 数据增量

- 产品型号、型号版本和规格
- 产品实例/序列身份
- 数字档案可见级别
- 质保与售后权益
- 所有权声明、转移和争议锁
- 状况评估、折损和估值快照
- 部件实例和更换记录
- 翻新批次和翻新证明

现有 `items`、所有权历史、服务历史、状态日志、发布和订单需要迁移适配，而不是废弃。

## 4. 可见层级

- 公众摘要：型号、真实性、公开质检、是否翻新
- 当前所有者：完整个人使用与服务记录
- 授权维修商：当前维修任务所需记录
- 生产/品牌组织：质量追踪和召回所需数据
- 平台审计：按案件和职责读取

历史所有者身份、联系方式、地址和私密使用记录不随所有权转移。

## 5. 验收门禁

- 单件身份与生产批次/现有物品映射唯一。
- 所有权转移依赖有效订单或经验证交接。
- 维修和更换记录追加，不允许删除历史事实。
- 估值明确标注规则版本、输入和时间。
- 新品和旧物可使用同一发布、订单和所有权链。
