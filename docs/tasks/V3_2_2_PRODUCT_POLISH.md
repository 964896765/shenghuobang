# 生活帮 V3.2.2：核心功能与使用体验收口

## 基线与范围

- 基线：`main` / `v3.2.1` / `906fdcb`
- 分支：`codex/v3.2.2-product-polish`
- 仅完善用户可见的物品、置换、回收、消息、通知、图片与页面状态。
- 不接入正式支付、S3/MinIO、ClamAV、正式推送、短信实名，不做金额迁移或 Expo 大版本升级。

## 现状问题清单

### P0：流程不可用或数据错误

- “置换”只有发布标签，没有置换请求、接收/拒绝/取消、双方确认和完成入口。
- 物品发布没有图片上传与展示；编辑、重新上架和受控删除也未接通。
- 回收询价缺少用户取消和商家处理入口，流程无法从等待完整推进到完成。
- 列表查询失败会被当作空数据或详情不存在，用户无法区分网络错误与真实空状态。

### P1：功能不完整

- 发布表单缺少所在地区、交换意向、退出提醒和失败后保留/重试说明。
- 我的发布只有查看入口，无法直接识别当前可执行的下一步操作。
- 会话列表没有未读数；发送失败没有保留待发送内容和明确重试入口。
- 通知跳转类型不完整，过期或已删除目标没有友好提示。
- 主要列表缺少下拉刷新，操作返回后容易看到旧状态。

### P2：页面与交互体验

- 物品卡片和详情没有图片失败占位；长文本、加载、错误提示不统一。
- 提交按钮虽可显示加载状态，但部分二次点击、失败反馈和成功反馈不统一。
- 键盘弹出、列表底部留白、确认弹窗与页面级重试需要统一处理。

## 实施优先级

1. 打通物品发布、编辑、图片、上下架和删除。
2. 补齐置换与回收的可执行状态流，并以服务端状态为准。
3. 完善聊天、通知及主要页面的加载、空、错和刷新状态。
4. 只为本次新增的核心判断与状态转换补充聚焦测试，保留全部历史测试。
