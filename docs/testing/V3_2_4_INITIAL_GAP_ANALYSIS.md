# V3.2.4 初始差距分析

## 1. 说明

- 本文档基于当前代码、自动化测试、阶段文档和已完成的 Android 真机只读证据整理。
- 本文档只描述“当前已知事实”和“待验证差距”，不把未执行真机或写操作验证的能力直接判定为通过。
- FUTURE、NEXT_RELEASE 与 OUT_OF_SCOPE 范围不视为 V3.2.4 当前缺陷。

## 2. 已实现能力

- 后端业务基础较完整：
  - 需求、报价、项目、里程碑、订单、支付、退款、投诉、认证、消息、通知、物品、回收、置换均已具备实体模型与状态机基础。
- 自动化覆盖较强：
  - 已有单测、MySQL 集成、运行时校验、空库迁移校验、Seed 幂等校验，可作为后端能力与数据一致性的强证据。
- 前端主路由基本齐全：
  - 已具备首页、发现、发布、消息、我的、登录、搜索、需求、项目、物品、回收、置换、订单、认证、后台等页面结构。
- 移动端准备已具备基础：
  - Expo SDK 54、Push Token 生命周期、权限处理、局域网诊断与 Android 返回键相关代码和测试基础已存在。
- 外部服务边界清晰：
  - 沙箱支付、开发级 Push、本地存储、开发级扫描策略已在文档和代码层有明确边界，不冒充生产接入。

## 3. 部分实现能力

- 需求发布与履约：
  - 已有完整页面和服务端流程，但前端多步骤写操作、报价接受到项目创建、项目履约主链路仍缺少系统性真机验证。
- 物品发布与管理：
  - 创建、编辑、图片上传、生命周期查看均有代码基础，但图片权限、上传稳定性、编辑回显和管理链路仍需真机写操作审查。
- 置换与回收：
  - 后端与页面均已具备主链路，但前端完整闭环的真实验证不足。
- 消息与通知：
  - 列表、详情、未读、通知和 Push Token 链路已存在，但会话详情、通知跳转、写消息等尚未做完整真机验证。
- 设置、权限与弱网反馈：
  - 已有设置页、权限入口和基础提示，但跨页面统一性仍待审查。

## 4. 当前问题与待确认项

### ANDROID-UI-001

- 问题：`我的`页“认证中心”左侧图标疑似缺失。
- 证据来源：已完成的 Android 真机只读巡检。
- 当前判断：属于 UI 一致性缺陷，优先级建议为 P2，如进一步影响识别或点击区域则上调为 P1。
- 仍需确认：
  - 是否仅 Android 显示缺失。
  - 是否为图标资源未渲染、颜色对比不足或布局挤压造成。

### ANDROID-UI-002

- 问题：搜索页点击热门搜索后结果为空。
- 证据来源：已完成的 Android 真机只读巡检。
- 当前状态：待确认。
- 根因确定程度：需要进一步定位。
- 严重级别候选：P1 / P2。
- 当前判断：目前只能确认“热门词触发后结果为空”，还不能直接写成已确认 P1 Bug。
- 仍需确认：
  - 同一热门词在 Web 或本地接口是否也为空。
  - 是否应该展示“无结果”还是当前页面表现不清晰。

## 5. 尚未验证

- 登录恢复、退出登录与重新登录。
- 首页、发现页的真机巡检。
- 发布需求、发布物品、图片上传、编辑管理的写操作验证。
- 置换、回收、订单的完整前端闭环。
- 消息详情页、通知跳转、会话写入。
- 权限拒绝、弱网、失败重试、多页面四态统一性。
- Android 长页面安全区与更多页面的返回键一致性。
- 历史 `Pressable transform` 崩溃修复后的全路径复验。

## 6. 后续功能

- 赠送、捐赠、租借、拍卖、新品共创属于后续扩展业务。
- 正式支付、金额迁移、正式 Push、正式对象存储、正式病毒扫描属于下一阶段或明确超出当前范围的能力。
- 这些事项应记录在路线中，而不是登记为 V3.2.4 当前缺陷。

- 版本基线冲突：
  - `README.md` 将稳定基线描述为 V3.2.2。
  - `AGENTS.md` 明确稳定版本为 V3.2.1。
  - 当前已统一为“稳定工程基线看 V3.2.1，功能闭环看 V3.2.2，移动端准备看 V3.2.3，V3.2.4 做审查与验收统一”。

- Seed 能力冲突：

  - 当前代码、README 和测试已表明 Seed 可重复执行。

## 8. 当前最可能的风险

### P0 风险

- 当前已确认 P0：0

### 环境问题

- 局域网 IP 变化：
  - 属于环境风险，不直接计为产品缺陷。
  - 仅当产品反馈不清楚时，才转化为体验问题。

### 潜在回归风险

- `Pressable transform`：
  - 属于历史修复后的回归关注项。
  - 当前不是新确认缺陷，但必须纳入无崩溃复验范围。

### P1 风险

- 搜索链路风险：
  - 搜索热门词结果为空，但在未确认数据契约、种子数据和空态设计前，暂只作为高关注待确认项。
- 发布与管理风险：
  - 物品发布、图片上传、需求发布、置换、回收等主写链路缺少系统性真机验证。
- 页面四态风险：
  - 首页、发现、订单、消息、设置等页面的加载、空态、失败态表达可能不统一。
- Android 交互风险：
  - 返回键、键盘、安全区和长页面滚动还未完成跨页面审查。

### P2 风险

- 视觉一致性风险：
  - “我的”页认证中心图标、菜单密度、列表对齐和图标规范仍存在明显待统一项。
- 运营后台体验风险：
  - 后台功能具备，但筛选、层级和可读性更偏工程实现，后续易在审查中暴露体验差距。
