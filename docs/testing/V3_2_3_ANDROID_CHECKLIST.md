# V3.2.3 Android 真机验收清单

## 当前执行状态

本清单必须在 Android 真机的 EAS development build 中人工执行。当前 Codex 环境没有 EAS 账号、构建凭据或 Android/iOS 真机，所以下列项目均未勾选；这不是失败，也不代表已经通过。执行者应记录设备型号、Android 版本、网络、build SHA、时间和证据。

- [ ] 记录设备型号、Android 版本、build SHA、执行人和日期
- [ ] development build 可安装，冷启动、热启动和前后台切换正常
- [ ] 手机与电脑处于同一局域网，API/WS 使用电脑 IPv4 而非 `localhost`
- [ ] 登录、会话恢复、退出登录和再次登录正常

## 核心页面与业务

- [ ] 首页、发现、搜索、订单、消息、通知和“我的”可打开与刷新
- [ ] 发布物品、保存草稿、编辑物品和未保存退出确认正常
- [ ] 相册选择、最多 6 张、排序、删除、上传失败重试正常
- [ ] 发起/处理置换、回收询价/报价、订单回看正常
- [ ] 聊天历史分页、发送、失败重试和未读刷新正常
- [ ] 通知点击可进入目标页面，目标已失效时有可理解提示

## 权限、键盘与布局

- [ ] 首次启动不主动请求通知或相册权限
- [ ] 点击选择图片时才请求相册权限
- [ ] 相册拒绝、永久拒绝和系统设置返回流程提示正确
- [ ] 通知允许、拒绝、永久拒绝和系统设置返回流程提示正确
- [ ] 登录、发布、编辑、聊天输入框不会被软键盘遮挡
- [ ] Android 返回键在未保存表单中先提示，在普通页面中按预期返回
- [ ] Modal、底部操作和 Toast 不被导航栏或安全区遮挡
- [ ] 小屏、横竖屏切换和字体放大后核心操作仍可触达

## 网络与 Push

- [ ] API 地址缺失、`localhost` 或不可达时显示明确诊断，不出现无限等待
- [ ] 弱网/断网时读取失败可重试，写操作不会自动重复提交
- [ ] WebSocket 断开后降级轮询，恢复网络后可重新连接和补拉
- [ ] development build 可获取 Expo Push Token 并在登录后注册
- [ ] App 前台收到通知时站内列表与未读数刷新
- [ ] 后台和冷启动点击 Push 可进入统一目标路由
- [ ] Token 更新后服务端记录更新，退出登录后当前 Token 停用
- [ ] 无效 Token 被服务端停用，推送失败不回滚业务事务

## iOS 对应检查

iOS 需要另行使用真实 development build 验证有限照片权限、系统设置返回、Safe Area、键盘、通知前台展示、后台/冷启动路由和 APNs 凭据。本仓库当前仅完成 SDK 54 代码与 bundle 兼容检查，不能用本清单代替 iOS 真机验收。
