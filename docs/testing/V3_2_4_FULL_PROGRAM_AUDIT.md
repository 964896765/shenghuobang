# V3.2.4 全程序审查结论

## 结论

V3.2.4 的代码范围、自动化范围和已知问题已经统一冻结。位置 R2、数据库迁移、发现/搜索/工程师位置体验和两个定位 P2 修复保留；旧临时隧道构建不再作为 RC 继续修补。当前代码可以进入 PR/CI 和受控 RC 验收，但在全部硬门禁关闭前仍是 `RELEASE BLOCKED`，不是正式发布批准。

## 已审查范围

- 认证、账号切换、角色与后端权限边界。
- 需求、报价、项目、里程碑、订单、沙箱支付、退款、托管、结算和投诉状态机。
- 物品、发布、购买、议价、置换、回收、所有权与并发保护。
- 消息、WebSocket、通知、文件、存储、审计、健康检查和迁移。
- V3.2.4-R2 前台位置权限、账号隔离、近似坐标、附近排序和无位置回退。
- Android 验证记录、仓库清理、依赖审计、构建与源码交付边界。

## 本轮关闭的代码问题

- 升级受影响的直接依赖并锁定安全传递依赖，生产依赖审计为零。
- 账号变化时取消请求并清空 TanStack Query 缓存，位置内存状态、缓存和异步结果按账号隔离。
- 有坐标的真实近似距离优先于只有“同地区”标签的历史数据。
- ESLint 警告作为失败处理并清除现有警告。
- 源码包只允许从干净 `HEAD` 生成，排除密钥、依赖、产物、日志和二进制包。
- CI 增加依赖审计、产品规范、Markdown、Expo 依赖和 Android JS bundle 检查。

## 冻结的外部硬门禁

- 在 GitHub 创建 PR，并让最新提交的完整 CI 通过。
- 在可用 MySQL 8 环境运行全部迁移、Seed 与集成脚本。
- 用户批准 R2 规范冻结；现有 `v3.2.4-spec-freeze` 不移动。
- 使用稳定测试 API 生成一份可安装、可长期复验的最终 RC APK。
- 在该最终 RC 上完成并归档 21 项真机冒烟。

这些项目需要 GitHub、MySQL、稳定测试环境或真机，不通过重复临时公网隧道解决。它们不阻止下一阶段在独立分支开发，但继续阻止 V3.2.4 正式发布。
